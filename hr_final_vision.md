## أولاً: إعدادات الشركة — نقطة التحكم المركزي

### إعدادات الامتثال القانوني (مفاتيح تشغيل/إيقاف)

> [!NOTE]
> الشركة غير مسجلة في التأمينات حالياً — كل الإعدادات القانونية موقوفة بالافتراضي. يُفعَّل كل مفتاح منفرداً متى أراد المدير.

| الإعداد | الافتراضي | ما يحدث عند التفعيل |
|---|---|---|
| التأمين الاجتماعي | موقوف | خصم ١١ بالمئة من الموظف وتوليد تقرير صاحب العمل ١٨.٧٥ بالمئة |
| ضريبة كسب العمل | موقوف | تطبيق الشرائح التصاعدية تلقائياً |
| التأمين الصحي | موقوف | خصم قيمة التأمين الصحي |

### إعدادات قواعد وقت العمل

- بداية يوم العمل الرسمي
- ساعات العمل القانونية اليومية (افتراضي ثماني ساعات)
- يوم العطلة الأسبوعية الافتراضي للشركة (الجمعة في مصر)
- نسبة الأوفرتايم: عادي مئة وخمسة وثلاثون بالمئة، عطلة رسمية مئة وسبعون بالمئة
- هامش التأخير المسموح قبل بدء الجزاء (بالدقائق)
- الحد الأقصى لأذونات الانصراف المبكر الشهرية (مرتان افتراضياً)

### إعدادات السلف

- الحد الأقصى لمبلغ السلفة (افتراضياً لا يتجاوز راتب شهر كامل)
- الحد الأقصى لعدد السلف النشطة (سلفة واحدة افتراضياً)
- أقل عدد أشهر خصم مسموح (شهر واحد)
- أعلى عدد أشهر خصم مسموح (ستة أشهر افتراضياً)

## ثانياً: جداول الإجازات والعطل — مرونة كاملة

### العطل الرسمية

- جدول مستقل قابل للإضافة والتعديل من الإدارة
- العطل تُدرج مسبقاً لكل سنة
- النظام يراعيها تلقائياً في الحضور والرواتب

### العطلة الأسبوعية — لكل موظف بشكل مستقل

```
حقل "يوم_العطلة_الأسبوعية" في ملف الموظف:
  فارغ (NULL) → يتبع إعداد الشركة (مثلاً الجمعة)
  قيمة محددة (مثلاً الأحد)
      → النظام يحترم عطلته الشخصية
      → لو غاب الجمعة وعطلته الأحد → يُسجَّل غياباً فعلياً
```

## ثالثاً: نظام الحضور الجغرافي (GPS)

### إعداد مناطق الحضور المسموحة

لكل فرع أو موقع:
- اسم الموقع وإحداثياته (تُختار من خريطة تفاعلية)
- دائرة السماحية بالأمتار (خمسون متراً للمكتب، مئتا متر للمستودع)
- هل يشترط صورة سيلفي؟ (نعم/لا)
- هل مفعَّل؟ (يمكن تعطيل موقع مؤقتاً)

الموظف المتنقل: نمط خاص يسمح له بالتسجيل من أي موقع معتمد.

### السيناريو: نسيان تسجيل الانصراف

```
الموظف يغادر النطاق بدون تسجيل انصراف
    ↓
إشعار فوري على هاتفه: "لم تسجل انصرافك بعد"
    ↓
(بعد ١٥ دقيقة خارج النطاق — قابلة للتعديل)
انصراف تلقائي مُعلَّم: "بانتظار مراجعة المشرف"
    ↓
المشرف يعتمد أو يُعدِّل الوقت
```

### السيناريو: الموظف المكتبي خرج بدون إذن

```
غادر النطاق خلال ساعات العمل
    ↓
إشعار فوري للمشرف: "الموظف [الاسم] خرج الساعة [X]"
    ↓
بإذن سابق  → يُسجَّل إذن انصراف مبكر
بدون إذن   → يُسجَّل "خروج غير مرخص" في الملف التأديبي
```

### السيناريو: المندوب بدون خطة زيارات

```
سجّل حضوراً وغادر بدون خطة يوم معتمدة
    ↓
تنبيه للمشرف: "المندوب يدوّر ميدانياً بدون خطة"
← ربط مع موديول خطط الزيارات (يُطور لاحقاً)
```

### السيناريو: انقطاع الإنترنت

- التطبيق يحفظ الإحداثيات والوقت محلياً
- عند عودة الاتصال يُزامَن تلقائياً بعلامة "تسجيل بدون اتصال"
- المشرف يراه بلون مختلف للمراجعة

## رابعاً: نظام الجزاءات المرن — محرك القواعد

### هيكل أي قاعدة جزاء

- النوع: تأخير، غياب، انصراف مبكر، خروج غير مرخص
- النطاق الزمني: من كم دقيقة إلى كم دقيقة
- الخصم: كسور يوم (ربع، نصف، يوم كامل)
- عدد المرات المسموحة قبل التطبيق
- التصعيد التلقائي بعد تجاوز الحد

### جدول الجزاءات الافتراضي (مُستخلَص من home-care migration 210 + أبحاث 2025)

جزاءات التأخير:

| المستوى | نطاق التأخير | الجزاء | السماح |
|---|---|---|---|
| فترة السماح | ١ - ١٥ دقيقة | لا جزاء | ثلاث مرات/شهر |
| تأخير بسيط | ١٥ - ٣٠ دقيقة | إنذار (كل ثلاثة إنذارات = خصم ربع يوم) | — |
| تأخير متوسط | ٣٠ - ٦٠ دقيقة | خصم ربع يوم | — |
| تأخير كبير | ٦٠ - ١٢٠ دقيقة | خصم نصف يوم | — |
| غياب جزئي | أكثر من ١٢٠ دقيقة | خصم يوم كامل | — |

جزاءات الغياب بدون إذن:

| الحالة | الجزاء |
|---|---|
| يوم غياب أول | خصم اليوم من الراتب |
| ثلاثة أيام متواصلة | خصم + تنبيه للمدير |
| أربعة أيام أو أكثر في الشهر | فتح قضية تأديبية تلقائياً |
| خمسة أيام متواصلة | تنبيه طارئ للمدير العام + إجراء قانوني |

الانصراف المبكر:

| الحالة | الجزاء |
|---|---|
| بإذن معتمد | خصم الوقت فقط |
| بدون إذن | خصم الوقت ضعفاً |
| تجاوز الحد الشهري للأذونات | تُعامَل كبدون إذن تلقائياً |

## خامساً: منظومة الإجازات — الصورة الكاملة

### أنواع الإجازات

| النوع | الرصيد | يُخصم من | يحتاج موافقة |
|---|---|---|---|
| سنوية | ٢١ - ٣٠ يوماً حسب سنوات الخدمة | رصيد الإجازة | نعم |
| عارضة | ستة أيام مخصومة من السنوية | رصيد الإجازة | لاحقاً |
| مرضية | حسب اللوائح مع شهادة طبية | رصيد مستقل | بعد التوثيق |
| أمومة | تسعون يوماً مدفوعة | مستقل | نعم |
| مناسبات زواج/وفاة | ثلاثة أيام مدفوعة | لا يُخصم | إشعار فقط |
| رسمية أعياد | من جدول الشركة | لا يُخصم | تلقائي |
| بدون أجر | بلا حد مع موافقة | يؤثر على الراتب | نعم |

### تدفق طلب الإجازة — كل الحالات

```
الموظف يقدم طلباً
    ↓
هل الرصيد كافٍ؟
    لا  → رفض تلقائي مع توضيح الرصيد المتبقي
    نعم → هل اليوم عطلة رسمية؟
              نعم → رفض: "هذا يوم عطلة أصلاً"
              لا  → هل يوجد تعارض مع إجازات الفريق؟
                        نعم → تنبيه للمشرف: "تعارض محتمل"
                        لا  → إشعار للمشرف
                              ↓
                          المشرف يعتمد أو يرفض
                              ↓
                          إشعار فوري للموظف
                              ↓
                          إذا اعتُمد: خصم من الرصيد + تحديث الحضور
```

## سادساً: نظام السلف — نوعان أساسيان

### النوع الأول: السلفة الفورية

- تُخصم كاملة من راتب الشهر الحالي
- الحد الأقصى نصف الراتب الشهري (قابل للتعديل)

### النوع الثاني: السلفة المجدولة (القرض)

- المبلغ الإجمالي + عدد الأشهر + قيمة القسط الشهري
- جدول أقساط مُسجَّل لكل شهر
- يُخصم القسط تلقائياً من مسير الرواتب كل شهر

### قواعد السلف

- لا يمكن الحصول على سلفة ثانية قبل سداد الأولى (إلا باستثناء إداري مُسجَّل)
- إذا انتهت خدمة الموظف: المتبقي يُستقطع من مستحقاته

### تدفق طلب السلفة

```
الموظف يختار النوع + المبلغ + عدد الأشهر
    ↓
تحقق تلقائي: هل يتجاوز الحد؟ هل لديه سلفة نشطة؟
    ↓
إشعار للمشرف → يعتمد أو يرفض
    ↓
إشعار لمدير HR → يعتمد أو يرفض
    ↓
إشعار للمالية للصرف الفعلي
    ↓
النظام يُنشئ جدول الأقساط تلقائياً
```

## سابعاً: الصلاحيات الديناميكية مع التفويض

### مبدأ التصميم

الصلاحية مرتبطة بالدور لا بالشخص. مدير HR يستطيع:
- منح أي دور صلاحية محددة مع تحديد نطاقها (فريق، فرع، فترة)
- سحبها في أي وقت

### جدول الصلاحيات الافتراضي

| الإجراء | الموظف | المشرف | مدير HR | المدير المالي | المدير العام |
|---|---|---|---|---|---|
| تسجيل حضوره | نعم | نعم | نعم | — | — |
| طلب إجازة | نعم | نعم | نعم | — | — |
| اعتماد إجازة فريقه | — | نعم | نعم | — | نعم |
| طلب سلفة | نعم | نعم | نعم | — | — |
| اعتماد سلفة فورية | — | يقترح | نعم | — | — |
| اعتماد سلفة مجدولة | — | — | نعم | نعم | — |
| تسجيل مخالفة تأديبية | — | يقترح | نعم | — | — |
| اعتماد مسير الرواتب | — | — | يراجع | يعتمد | — |
| إنهاء خدمة موظف | — | — | يقترح | — | يعتمد |
| تعديل إعدادات الشركة | — | — | نعم | — | نعم |

### آلية التفويض الزمني

```
مشرف أو مدير في إجازة:
    → يختار مفوَّضاً من نفس المستوى أو أعلى
    → يحدد مدة التفويض (من تاريخ لتاريخ)
    → النظام ينقل إشعاراته وقراراته للمفوَّض تلقائياً
    → عند انتهاء المدة: يعود تلقائياً دون تدخل
    
كل تفويض مُسجَّل في سجل الأحداث: من فوَّض من، من متى لمتى، ما الصلاحية
```

## ثامناً: ربط المستخدم بالموظف — بلا تعقيد

### المشكلة

لدينا:
- المستخدم (User): حساب في Supabase Auth له دور وصلاحيات
- الموظف (Employee): سجل له راتب وحضور وإجازات

### الحل: الربط شبه التلقائي

```
عند إنشاء ملف موظف جديد:
    حقل البريد الإلكتروني في ملف الموظف
        ↓
    البحث عن مستخدم بنفس الإيميل في Auth
        ↓
    وجد   → ربط تلقائي (user_id في جدول الموظفين)
             رسالة تأكيد: "تم الربط بحساب [الإيميل]"
        ↓
    لم يجد → ثلاثة خيارات:
              أولاً:  إنشاء حساب جديد (دعوة بريدية)
              ثانياً: موظف بدون حساب (عمالة يدوية)
              ثالثاً: ربط يدوي لاحقاً من قائمة المستخدمين
```

### قاعدة المعرّف الدائم

كل موظف له رقم موظف ثابت لا يتغير حتى لو تغيّر إيميله أو اسمه أو دوره أو عاد بعد إنهاء خدمته. هذا الرقم يربط كل سجلاته التاريخية.

### عند إنهاء الخدمة

```
إنهاء الخدمة مُعتمَد من المدير العام
    ↓ تلقائياً وفوراً:
    - إيقاف حساب المستخدم في Auth
    - الاحتفاظ بكل السجلات (لأغراض التدقيق)
    - حساب المستحقات النهائية (راتب + سلف + نهاية الخدمة)
```

## تاسعاً: كشف الحساب اللحظي للموظف

ما يرى الموظف عن نفسه في أي لحظة دون الحاجة لسؤال HR:

```
حسابي الشهري — مارس ٢٠٢٦

المستحقات:
  الراتب الأساسي حتى اليوم    ٣٥٠٠.٠٠
  بدل المواصلات                 ٥٠٠.٠٠
  بدل السكن                     ٨٠٠.٠٠
  أوفرتايم                      ٢٤٠.٠٠
  عمولات محصَّلة حتى الآن       ٩٨٠.٠٠
  إجمالي المستحقات             ٦٠٢٠.٠٠

الخصومات:
  خصم الغياب (يوم)             ١٦٦.٠٠
  خصم التأخيرات                  ٤٢.٠٠
  قسط السلفة                    ٥٠٠.٠٠
  إجمالي الخصومات               ٧٠٨.٠٠
```

### التبويبات الأخرى للموظف

- حضوري: جدول شهري بألوان (أخضر حضور، أصفر تأخير، أحمر غياب، رمادي عطلة)
- هدفي: للمندوبين فقط — الهدف، المحصَّل، نسبة الإنجاز، العمولة المتوقعة

## عاشراً: خريطة التكامل مع الموديولات

```
الموارد البشرية تأخذ من:
    موديول المبيعات    → حجم مبيعات كل مندوب
    موديول التحصيلات  → المبالغ المحصلة فعلاً (قاعدة العمولة)
    موديول التوصيل    → رحلات السائق (مؤشرات الأداء)
    موديول المخزن     → دقة صرف المخزني
    موديول الزيارات   → خطة الزيارات للمندوب

الموارد البشرية تعطي لـ:
    موديول المالية    → قيد راتب تلقائي لكل مركز تكلفة عند الاعتماد
    نظام الصلاحيات   → الوظيفة تُحدد قائمة الصلاحيات
    نظام الإشعارات   → كل أحداث HR تولِّد إشعارات ذكية
```

### قواعد التكامل الذهبية

- قاعدة الاتجاه الواحد: بيانات الأداء تأتي من الموديولات فقط — لا ترجع
- قاعدة النقطة الواحدة للحقيقة: رقم التحصيل في موديوله هو المرجع — لا يُدخَل يدوياً في HR
- قاعدة التزامن الفوري: إنهاء خدمة موظف = إيقاف وصوله تلقائياً وفوراً
- قاعدة مراكز التكلفة: القيد المحاسبي يُوزَّع على قسم وفرع تلقائياً

## ثاني عشر: ما استخلصناه من مشروع home-care

### ميزات مطوَّرة من home-care لإيدارة

| الميزة في home-care | ما سنبنيه في إيدارة |
|---|---|
| advance_type فوري أو أقساط | نفس المفهوم + واجهة أوضح + قيود صارمة |
| جدول penalty_rules مرن | نفسه + أنواع انصراف مبكر وخروج غير مرخص |
| auto_check_out عند إنهاء الرحلة | Trigger انصراف تلقائي للسائقين |
| جدول public_holidays منفصل | نفسه + ربط مرن باليوم الأسبوعي لكل موظف |
| late_minutes في سجل الحضور | نفسه + حساب جزاء متدرج |
| paid_leave_days لكل عامل | نفسه + ربط بنوع التعاقد |
| termination_date مع pro-rata | نفسه + تسوية نهاية الخدمة للمالية |
| company_locations بإحداثيات وسماحية | نفسه + إشعار فوري عند الخروج |

### ميزات إيدارة الإضافية التي لا توجد في home-care

- العطلة الأسبوعية المرنة لكل موظف بشكل مستقل
- صلاحيات ديناميكية مع تفويض زمني وسحب تلقائي
- ربط المستخدم بالموظف شبه تلقائي بالإيميل
- نظام عمولات مرتبط بلائحة أهداف
- كشف حساب لحظي تفاعلي للموظف
- تنبيه الخروج الجغرافي غير المرخص
- ربط خطط الزيارات بالحضور الميداني
- التأمينات والضرائب قابلة للتفعيل والإيقاف
